# ハイレベルデザイン: input_image（レシート画像入力）

## この機能は何か

レシート画像を Claude API（vision）で解析し、家計簿データとしてフォームに反映する機能。画像の取得・圧縮・AI 解析・フォーム反映・複数レシート一括送信までを担う。スプレッドシートへの書き込み自体は共通の送信処理に委譲する。

**責務外:** スプレッドシートの列マッピング、税計算の詳細ロジック、フォームの UI 構造（これらは voice_input と共有する共通基盤の責務）。

## 概念モデル

- **レシート画像** — ユーザーが撮影/選択した画像ファイル。JPEG 圧縮後に AI 解析と Drive アップロードの 2 つの用途に使用される | 既存
- **解析結果（レシートデータ）** — AI が画像から抽出した構造化データ。date, payer, memo, amount, categories, calcExpressions を持つ JSON オブジェクト | 既存
- **レシートキュー** — 1 画像から検出された複数レシートデータの配列。現在表示中のインデックス（`currentReceiptIndex`）で状態管理する | 既存
- **カテゴリ分類記号** — ユーザーがレシートに手書きする記号とカテゴリの対応関係。AI プロンプト内に自然言語で定義されている | 既存

## 外部システムとの関係

### Claude API（vision）
- 画像 + システムプロンプト → 構造化 JSON 配列
- モデル: `claude-sonnet-4-20250514`、max_tokens: 1500
- ブラウザから直接呼び出し（`anthropic-dangerous-direct-browser-access` ヘッダー使用）
- API エラー時: エラー表示のみ。リトライなし

### Google Drive（via GAS）
- レシート画像のバックアップ保存
- フォルダ: `kakeivoice_receipts`（GAS 側で自動作成）
- ファイル名: `receipt_{timestamp}.jpg`
- アップロード失敗時もデータ入力には影響しない（非同期・非必須）

### voice_input（共通基盤）
- 借りるもの: `updateFormFromParsed()`（フォーム反映）、`sendSingleData()`（送信）、`updateTotals()`（税計算）、`validateForm()`（バリデーション）、`resetForm()`（フォームリセット）
- 渡すもの: レシートキューの状態（`receiptQueue`, `currentReceiptIndex`）

## 処理の全体像

### メインフロー: 画像入力 → 送信

1. ユーザーが「画像から入力」ボタンをクリック → ファイル選択ダイアログ
2. 画像を選択 → `compressImage()` で JPEG 圧縮（長辺最大 1600px、品質 0.85）
3. プレビュー表示 + 「画像解析中...」ステータス
4. **並行実行:** `parseImageWithClaudeAPI()` と `uploadImageToDrive()`
5. 解析結果をレシートキューに格納
6. 1 件目のレシートをフォームに反映（`loadReceiptFromQueue(0)`）
7. 複数レシートの場合: キューバッジ表示 + 送信ボタンに件数表示
8. ユーザーがフォーム内容を確認・編集
9. 送信ボタンで全件まとめて送信（`submitData()`）

### サブフロー: 複数レシートのナビゲーション

- 「◀ 前のレシート」「次のレシート ▶」ボタンでキュー内を前後に移動できる
- 送信ボタンは現在表示中の 1 件のみを送信する
- 送信成功後、キューに次のレシートがあれば自動的に次を表示する
- 最後のレシートを送信するとキューがクリアされフォームがリセットされる

### AI 応答のパース

- 正常な JSON 配列 → そのまま使用
- JSON として不正な場合 → 正規表現で `[...]` または `{...}` を抽出してパース
- 単一オブジェクトの場合 → 配列に変換（`[parsed]`）
- パース失敗 → null を返し、エラー表示

## 設計上の特徴

- **AI プロンプト駆動のルール定義:** カテゴリ分類・支払者判定・日付推定のルールは全て AI プロンプト内に自然言語で記述されている。コード側にはこれらのロジックを持たず、AI の出力をそのまま使用する設計
- **並行処理によるUX最適化:** AI 解析と画像アップロードを `Promise.all` で並行実行し、待ち時間を最小化
- **非必須の画像バックアップ:** Drive アップロードは失敗してもデータ入力フローをブロックしない。データの正確性よりも入力完了を優先する判断
- **フォールバック付き JSON パース:** AI の出力形式が不安定な場合に備え、正規表現による抽出を試みる

## 関連ケイパビリティとの境界

- **voice_input**: フォーム表示・送信・税計算・バリデーションの関数を借りる。AI プロンプトの内容と画像処理は input_image 固有。両者は同じフォームを共有するため、解析結果のデータ形式（date, payer, memo, amount, categories）は統一されている
- **settlement_dashboard**: 直接の接点なし。input_image が送信したデータはスプレッドシート経由で間接的にダッシュボードに反映される
